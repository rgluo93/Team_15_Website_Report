Weeks 15-16: 23/02 - 08/03

Focused on integration, internal testing, and deployment refinement; iterated on course content, UX, and LangGraph workflows based on client requirements.

<!--truncate-->

This week, we made major improvements to the LangGraph workflow. In order to fulfill the feature requirements for the education officer chatbot, we added a coaching tool to the workflow, which is responsible for generating structured outputs specific to education officer-related tasks. We made use of Pydantic schemas and custom prompts to make sure the LLM responds in a consistent and predictable manner for requests like visit plans and zone reviews. Finally, we connected the WhatsApp client to the LangGraph agent to allow the workflow to be accessible via WhatsApp messaging, bringing everything together. 

In terms of the frontend, we also made the LangGraph agent accessible via a chatbot window when logged in as an education officer, and fully implemented the speech-to-text feature. 
As for deployment, we successfully Dockerized and deployed it on Azure as a container app. From then on, we continuously updated the Azure instance with new changes we made in order to test our features. We also did expansive unit testing during this period, mainly for RAG, LangGraph agent and ChatService.
