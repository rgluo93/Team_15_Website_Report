Weeks 13-14: 09/02 - 22/02

We delivered key frontend features including the chatbot window, resolved WhatsApp integration issues, and began building the LangGraph agent and MCP server integrations.

<!--truncate-->

In early February, we started the frontend development process. However, since the existing codebase was given to us by the Dignitas team, we had a multitude of problems accessing, editing or running the frontend. We had a lengthy meeting on 11/2 to set up FTP access to edit files. Even when working in the devspace, we had to be careful as the codebase was maintained by the Dignitas team, making sure to document all changes clearly. That week, we added the chatbot window to the website, allowing a user-friendly interface for the AI. 

We also faced problems with WhatsApp, since Meta blocked us from sending messages for being unverified. We conducted another lengthy meeting with the Dignitas team to tie the developer app to their Meta Business account, then re-configured the WhatsApp client to reflect this change. We also refined the client to be more robust, handling requests and disconnect errors more cleanly.

During this period, we also got started on the most complex feature, the LangGraph agent. We crafted the initial flow of the agent, which involved being able to classify messages and call the tools WhatsApp, SQL and translation. We then set up the MCP server to host the above tools and connected it to the workflow.
